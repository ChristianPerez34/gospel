## Why this matters

Every discovered `SKILL.md`'s YAML frontmatter is parsed with the deprecated
`serde_yaml` crate. `Cargo.lock` resolves it to
`version = "0.9.34+deprecated"`. A deprecated, unmaintained parser owns the
input boundary for workspace and global skill content (user-influenced YAML
under `<workspace>/.agents/skills/` and the user's global data directory).

The replacement selected on 2026-07-21 is `yaml_serde` 0.10.4. Its current
docs describe it as the actively maintained fork of `serde_yaml`, published by
the official YAML organization, and expose the required Serde-compatible
`yaml_serde::from_str` API. `noyalib` 0.0.15 was also evaluated; it is active
and Serde-compatible, but its pre-1.0 API and broader YAML 1.2/security policy
surface make it a less conservative choice for this one-call-site migration.
`serde_yml` is not a target because its current release is deprecated and is a
compatibility shim.

## Steps

### Step 1: Swap the dependency

In `src-tauri/Cargo.toml`, replace `serde_yaml = "0.9"` with
`yaml_serde = "0.10"`. The selected crate exposes the same `from_str` entry
point used by this repository and is maintained by the YAML organization.

Remove the `serde_yaml` line and let cargo resolve `yaml_serde` (you will
re-add the line in the lockfile in the next step). Do not leave both
listed â€” `serde_yaml` must be removed entirely.

### Step 2: Update the import + call site

In `src-tauri/src/skills.rs:340` (and any other `serde_yaml::` reference
in the file â€” verify with `rg "serde_yaml" src-tauri/src`):

- Replace `serde_yaml::from_str(frontmatter_str)` with
  `yaml_serde::from_str(frontmatter_str)`.
- Replace any `use serde_yaml;` / `use serde_yaml::...` import with the
  equivalent `yaml_serde` import (the function name `from_str` is identical).
- Preserve the current `match` behavior: log the parser error through its
  `Display` implementation and skip the invalid skill. There is no repository
  `SkillParseError` conversion on this path.

### Step 4: Run the skills test cohort + the full backend suite


The full filtered skills cohort must pass (the migration is intended to be
