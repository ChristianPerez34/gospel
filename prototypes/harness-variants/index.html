Variant 01

Run Ledger Workbench

The app becomes a single continuous run record. Chat, tool calls, diffs, approvals, and tests live in one chronological ledger, with evidence pinned on the right. This is the closest to the current app, but clearer because there is no mode switcher to understand first.

  • Best for: current chat-first workflow with review visible but secondary.
  • Tradeoff: less dramatic than the other directions, but easiest to ship incrementally.
  • No tabs: review and pipeline become evidence cards attached to the active run.

Variant 02

Command Runway

The prompt becomes the center of gravity. Instead of selecting a layout, the user launches an intent into a visible sequence: context, plan, patch, verify, handoff. It feels more like a focused tool than a chat app.

  • Best for: users who want to steer the agent by jobs, not conversations.
  • Tradeoff: bigger product shift because chat becomes a transcript below the command surface.
  • No tabs: the run stages are ordered work packets, not alternate views.

Variant 03

Triage Cockpit

The harness is organized into three permanent lanes: intent, agent work, and evidence. This is the most explicit replacement for Pair and Pipeline because it shows what each surface is for without asking the user to switch layouts.

  • Best for: multi-agent review, approvals, and high-control sessions.
  • Tradeoff: denser and more operational; less calm for simple one-shot prompts.
  • No tabs: lanes stay visible and resize/collapse by intent, not by named modes.

Recommended direction

Safest next step: Run Ledger

It preserves Gospel's chat-first product model and can absorb current ReviewPanel work without a mode switcher.

Boldest product bet: Command Runway

It reduces chatbot feel and makes the harness task-native, but would require deeper information architecture changes.

Most powerful: Triage Cockpit

It is strongest for multi-agent and review-heavy sessions, but risks overwhelming simple coding work.
